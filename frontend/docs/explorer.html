Snapshot Explorer

The Explorer (tenant sidebar) lets you open any config snapshot and browse everything in it by category: applications, flows, policies, groups, bindings, and whatever else your provider exports. Pick a snapshot at the top - the newest one is selected by default.

Status badges

When you open an older snapshot, every object carries a badge comparing it to the latest backup:

  • unchanged - identical in the latest backup.
  • modified - exists in the latest backup with different configuration; the object detail lists exactly which fields changed and shows both versions side by side.
  • deleted in latest - present in this snapshot but gone from the latest backup. This is the one to look for after an accidental deletion.
  • new in latest - did not exist yet when this snapshot was taken.

Badges compare snapshots, so they work offline and instantly. Category tiles show a +N / -N delta against the latest backup for the same reason.

One-click object restore

Every object row has a Restore… button. It opens the restore dialog with a dry-run preview already running and only that object selected - one click to preview, one to apply, never a silent write. Dependencies and manual follow-ups (like credential regeneration) are flagged in the preview, exactly as in a full restore.

Write actions follow your role: org viewers can browse everything but see no Restore buttons, and the server enforces the same rule regardless.
